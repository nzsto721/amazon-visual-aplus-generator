# Amazon Image Requirements Reference

> 资料来源：Amazon Seller Central 官方产品图片要求（2026版）、Amazon A+ Content 图片规格、OpenAI GPT Image 2 API 文档。

---

## 一、主图（Main Image）完整规范

### 1.1 技术规格

| 要求 | 规范 |
|---|---|
| 最小尺寸 | 1,000 × 1,000 px（最长边） |
| 推荐尺寸 | 2,000 × 2,000 px 或更大 |
| 宽高比 | **推荐 1:1 正方形** |
| 最大文件大小 | **10 MB** |
| 可接受格式 | JPEG（推荐）、PNG、TIFF、GIF（非动画） |
| 色彩模式 | **sRGB** |

### 1.2 内容要求

**主图必须：**
- 展示所售的实际产品（不能是图形、插图或占位符）
- 产品至少占据画面的 **85%**
- 纯白背景（RGB 255, 255, 255）
- 展示完整产品——边缘不能被裁剪

**主图严禁包含：**
- 任何形式的文字、Logo 或水印
- 徽章、奖项或 "Best Seller" 标签
- 多个产品（捆绑销售套装除外）
- 道具、配件或未包含在购买中的物品
- 边框、色块或任何非纯白的背景
- 人体模特/模特架（服装类仅允许真人模特）

### 1.3 缩放（Zoom）功能

- 图片最长边达到 **至少 1,600 px** 才启用缩放
- 推荐 **2,000–3,000 px** 以获得最佳移动端缩放体验
- 启用缩放后，部分展示位置会获得优先权

## 二、副图（Secondary Images）完整规范

### 2.1 数量与显示

| 项目 | 规格 |
|---|---|
| 总数上限 | **9 张**（1 主图 + 8 副图） |
| 桌面端默认显示 | **7 张** |
| 推荐尺寸 | 1,200 × 1,600 px（竖版） |
| 最小尺寸 | 1,000 px（最长边） |
| 文件大小 | ≤ 10 MB |

### 2.2 副图允许包含的内容（与主图不同）

- 文字叠加和功能标注
- 带有功能亮点的信息图（infographic）
- 生活方式/使用场景照片
- 尺码表和尺寸标注
- 对比图（before/after 或 vs 竞品）
- 细节特写
- 包装内容物布局图

### 2.3 信息图（Infographic）最佳实践

- **限制在 4-5 个标注点**——过多会眼花缭乱
- **使用大号易读字体**——需在手机屏幕上测试可读性
- **保持品牌一致性**——全系列使用相同颜色、字体和风格
- **以利益点为引导，而非功能点**——"保冷 24 小时"优于"双层真空隔热"

### 2.4 尺寸标注图设计要点

- 包含关键尺寸及清晰标注线（长、宽、高、深）
- **双重单位**——同时标注英寸（in）和厘米（cm）
- **比例参照**——将产品放在可识别的物体旁（手、硬币、手机）
- 使用细而清晰的标注线，不遮挡产品
- 选择与产品形成对比的字体颜色
- 标注风格保持一致（相同的线宽、字体和颜色）
- 尽量将尺寸标注放在产品轮廓外部

### 2.5 推荐排列策略

| 位置 | 目的 | 示例 |
|---|---|---|
| 图1（主图） | 白底产品图 | 干净、完整的产品拍摄 |
| 图2 | 核心功能信息图 | 4-5 个带图标的标注 |
| 图3 | 尺寸和规格 | 在产品上标注尺寸 |
| 图4 | 生活方式/使用场景 | 产品在真实环境中的使用 |
| 图5 | 材质/细节特写 | 纹理、缝线、表面质量 |
| 图6 | 包装内容 | 盒内包含的所有物品 |
| 图7 | 对比或社交证明 | 与竞品对比，或客户使用案例 |

## 三、A+ Content（Enhanced Brand Content）完整规范

### 3.1 桌面版模块图片

| 规格 | 值 |
|---|---|
| 尺寸 | **1464 × 600 px** |
| 每页最大模块数 | **7 个** |
| 可接受格式 | JPEG、PNG |
| 文件大小建议 | ≤ 5 MB |
| 图片中文字 | 允许（且推荐用于品牌叙事） |

### 3.2 移动版模块图片

| 规格 | 值 |
|---|---|
| 尺寸 | **1600 × 1200 px** |
| 每页最大模块数 | **7 个** |
| 备注 | 模块数必须与桌面版一致；每个模块可使用不同图片 |

### 3.3 常见 A+ 模块类型

| # | 模块名称 | 说明 |
|---|---|---|
| 1 | Standard Company Logo | Logo + 简短标语 |
| 2 | Standard Image & Text Overlay | 全幅图片 + 文字叠加层 |
| 3 | Standard Single Image & Highlights | 一张图片 + 要点列表 |
| 4 | Standard Four Image & Text | 2×2 网格功能卡片 |
| 5 | Standard Single Left Image | 左侧图片 + 右侧文字 |
| 6 | Standard Single Right Image | 左侧文字 + 右侧图片 |
| 7 | Standard Product Comparison Chart | 对比表格（需手动 HTML 填写） |
| 8 | Standard Technical Specifications | 技术参数表（需手动 HTML 填写） |

### 3.4 Brand Story（品牌故事）

| 规格 | 值 |
|---|---|
| 桌面版尺寸 | 1464 × 600 px |
| 移动版尺寸 | 1600 × 1200 px |
| 卡片上限 | 19 张 |

## 四、常见被拒原因（Amazon 审核高频问题）

| # | 问题 | 原因 | 解决方案 |
|---|------|------|----------|
| 1 | 背景非纯白 | 视觉上像白色但实际不是 RGB(255,255,255) | 编辑器中填充纯白 #FFFFFF，用吸管工具验证 |
| 2 | 产品未占画面 85% | 产品周围空白过多 | 更紧密裁剪，产品应几乎触及画面边缘 |
| 3 | 主图上有文字或图形 | 品牌 Logo、尺码标签或促销文字 | 所有文字移至副图 |
| 4 | 水印或摄影师署名 | 工作室水印或版权文字 | 索要无水印文件或后期移除 |
| 5 | 服装图可见模特架 | 服装展示在可见人体模特架上 | 使用隐形模特架技术或真人模特 |
| 6 | 分辨率过低 | 图片 < 1,000 px，模糊无法缩放 | 以更高分辨率重新拍摄 |
| 7 | 展示多个产品（非套装） | 主图含未包含在购买中的配件 | 仅展示买家收到的内容 |

## 五、按类别的额外规则

### 宠物用品（重点）
- 主图：产品本身，白色背景
- 副图：**尺码/适用体重/适用品种信息至关重要**
- 材质安全认证可在副图中展示（需有证明）
- 使用场景图必须真实——不得虚构使用效果
- 食品/零食类必须展示成分标签或营养信息

### 服装
- 禁止模特架，必须真人模特或平铺拍摄
- 至少一张背面图
- 强烈建议添加尺码表

### 电子产品
- 展示所有包含的配件和线缆
- 端口和连接特写
- 避免过度 CGI——优先实物照片

### 家居家具
- 包含房间尺度的生活方式图
- 尺寸标注至关重要（长×宽×高）
- 展示组装结果，非仅盒中零件

### 食品
- 必须展示在运输包装之外
- 可能需要营养标签图片
- 新鲜食品须真实但令人有食欲

## 六、GPT Image 2 生成约束（OpenAI API）

> 资料来源：OpenAI Developer Docs — Image Generation API

### 6.1 四大约束（必须全部满足）

| 约束项 | 规则 |
|---|---|
| **最大边长** | 最长边 ≤ **3,840 px** |
| **16 倍数** | 宽和高**都必须**是 16 的整数倍 |
| **长宽比上限** | 长边:短边 ≤ **3:1** |
| **总像素范围** | 介于 **655,360** 至 **8,294,400** 之间 |

### 6.2 质量设置

| 等级 | 适用场景 |
|---|---|
| `low` | 草稿、提示词探索阶段 |
| `medium` | 构图确认、创意审阅（推荐默认值） |
| `high` | 最终定稿候选 |

**推荐工作流**：小尺寸 + 低质量快速迭代 → 确认构图 → 高质量 + 目标尺寸生成最终稿。

### 6.3 其他约束

| 项目 | 说明 |
|---|---|
| 输出格式 | PNG（默认）、JPEG、WebP |
| 透明背景 | **不支持**——GPT Image 2 无法生成透明背景 |
| 4K 输出 | 超过 2560×1440 为实验性功能，更慢更不稳定 |

### 6.4 对 Amazon 尺寸的影响

| Amazon 尺寸 | 能否直接生成 | 原因 |
|---|---|---|
| 1200×1600（副图） | ✅ 可以 | 1200÷16=75，1600÷16=100，总像素 1,920,000 |
| 1464×600（A+桌面） | ❌ 不行 | 1464÷16=91.5，600÷16=37.5，都不整除 |
| 1600×1200（A+移动） | ✅ 可以 | 1600÷16=100，1200÷16=75，总像素 1,920,000 |
| 2000×2000（主图） | ✅ 可以 | 2000÷16=125，总像素 4,000,000 |

## 七、实际生成尺寸对照表（Skill 内部使用）

| 资源类型 | Amazon 上传尺寸 | 模型生成画布 | 裁切说明 |
|---|---|---|---|
| 副图（secondary） | 1200 × 1600 | 1200 × 1600 | 不需裁切，可直接生成 |
| A+ 桌面版（aplus_desktop） | 1464 × 600 | **1488 × 608** | 居中裁切至 1464×600（左右各裁 12px，上下各裁 4px） |
| A+ 移动版（aplus_mobile） | 1600 × 1200 | 1600 × 1200 | 不需裁切，可直接生成 |
| 品牌故事桌面（brand_story_desktop） | 1464 × 600 | **1488 × 608** | 同 A+ 桌面版 |
| 品牌故事移动（brand_story_mobile） | 1600 × 1200 | 1600 × 1200 | 不需裁切 |

## 八、上传前最终检查清单

### 主图检查
- [ ] 纯白背景（RGB 255, 255, 255）
- [ ] 产品占画面 ≥ 85%
- [ ] 无文字、Logo、水印、徽章
- [ ] 最长边 ≥ 1,600 px（推荐 2,000+）
- [ ] sRGB 色彩模式，文件 ≤ 10 MB
- [ ] 产品外观与实物一致（AI 未改动形状/颜色）

### 副图检查
- [ ] 信息图文字在移动端可读（拇指大小测试）
- [ ] 所有标注内容有事实依据
- [ ] 尺寸同时标注英寸和厘米
- [ ] 无虚假认证或未验证声明

### A+ 内容检查
- [ ] 桌面版 7 张（1464×600）+ 移动版 7 张（1600×1200）
- [ ] 所有模块图片尺寸合规
- [ ] 文字在两种尺寸下均可读
- [ ] 品牌故事叙事连贯（如有）

### 合规底线
- [ ] 无虚假认证（FDA、CE、RoHS 等需有证明）
- [ ] 无未验证的医疗/健康声明
- [ ] 无未验证的环保声明（"eco-friendly"、"biodegradable"）
- [ ] 无未验证的性能声明（"10x stronger"、"lifetime warranty"）
- [ ] 无竞品品牌名称或商标词
- [ ] 所有英文拼写和语法正确
